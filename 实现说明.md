## 架构设计原则

### 模块化设计
完全遵循文档中的四层架构：

```
Data Aggregator → Signal Generator (DeepSeek) → Risk Manager → Execution Engine
        ↓                                                              ↓
   Aster API ←──────────────────────────────────────────────────── Aster API
```

### 关键创新点

#### 1. **提示词即算法**
- 传统硬编码的交易逻辑被高度结构化的 LLM 提示词取代
- 提示词包含完整的分析框架和风险规则
- 可以通过修改提示词来调整策略，无需改代码

#### 2. **独立风险管理**
- Risk Manager 独立于 AI 核心运行
- 拥有否决权，可拒绝 AI 的任何信号
- 拥有紧急平仓权限，保护账户安全

#### 3. **数据驱动决策**
- 提供 100+ 根 K 线数据
- 多时间框架分析（3分钟 + 4小时）
- 15+ 种技术指标
- 资金费率和持仓量数据

## 风险管理实现

### 仓位大小计算

```python
风险金额 = 账户权益 × 风险百分比
价格风险 = |入场价 - 止损价| / 入场价
仓位名义价值 = 风险金额 / 价格风险
实际仓位 = min(仓位名义价值, 最大仓位限制)
```

### 五层保护机制

1. **信号级**: LLM 必须提供止损和置信度
2. **验证级**: Risk Manager 检查盈亏比 ≥ 1.5
3. **仓位级**: 动态计算仓位大小，限制单笔风险
4. **账户级**: 回撤超限自动平仓所有仓位
5. **监控级**: 实时检测清算风险
